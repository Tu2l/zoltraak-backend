AsciiDoc Syntax and Uses

Document Structure

Document Title

Starts at the beginning of the document with one equals sign =. Only one level 0 title is allowed.

= My Awesome Document

Section Titles

Use equals signs = to denote section levels. The number of equals signs corresponds to the heading level (1 to 6).

== Level 1 Section
=== Level 2 Subsection
==== Level 3 Sub-subsection

Preamble

The paragraph(s) immediately following the document title (level 0 heading) and before the first level 1 section. It’s typically rendered in a larger font and used to summarize the document.

= My Important Report
This report provides an overview of the project findings. It includes analysis and recommendations.

== Section One
...

Document Attributes

Defined in the document header using a colon : followed by the attribute name, a colon, and the value.

= My Book
:author: John Doe
:version: 1.0
:toc:
:numbered:
:imagesdir: path/to/images

  • :toc:: Enables the table of contents.

  • :numbered:: Numbers the sections.

  • :author: Value: Specifies the author.

  • :version: Value: Specifies the document version.

  • :imagesdir: path/to/images: Sets the default directory for images.

Comments

Lines starting with // are treated as comments and are not rendered.

// This is a comment.
== My Section // Another comment on the same line.

Blocks

AsciiDoc documents are composed of blocks separated by one or more blank lines. Different types of blocks have specific syntax.

Text Formatting (Inline Styles)

Bold

Enclose text in asterisks *.

This is *bold* text.

Italic

Enclose text in underscores _.

This is _italic_ text.

Monospace (Code)

Enclose text in backticks ` or double backticks ` `.

Use the `printf()` function.
The variable ``user_id`` stores the identifier.

Literal (Verbatim)

Enclose text in plus signs +. This prevents any formatting within the enclosed text.

This will display as +*literal*+.

Bold and Italic

Combine the syntax.

This is *_bold and italic_*.
This is **__also bold and italic__**.

Superscript

Enclose text in carets ^.

E = mc^2^

Subscript

Enclose text in tildes ~.

H~2~O

Highlight

Enclose text in hash marks #.

This is #highlighted# text.

Quoted Text

Use single or double quotes. Asciidoctor can be configured to use smart quotes.

'Single quoted text'
"Double quoted text"

Pass-through

Allows embedding raw HTML or other markup.

Single-line

Use + at the beginning and end.

+++<span style="color:red;">This is red text</span>+++

Multiline block

Use as delimiters.

++++
<table>
  <tr>
    <td>Row 1, Cell 1</td>
  </tr>
</table>
++++

Inline pass

Use text to pass text directly. You can also specify substitutions to disable within the pass: block (e.g., Don't smarten 'these' quotes).

pass:[inline text]
pass:quotes[Don't smarten 'these' quotes]

Lists

Unordered Lists (Bulleted)

Start each item with an asterisk * or a hyphen - followed by a space. Nesting is achieved by using more asterisks or hyphens.

* Item 1
* Item 2
** Sub-item 2.1
** Sub-item 2.2
* Item 3

- Another top-level item
-- A nested item

Ordered Lists (Numbered)

Start each item with a period . followed by a space. Nesting is done with more periods.

. Step 1
. Step 2
.. Sub-step 2.1
.. Sub-step 2.2
. Step 3

Description Lists (Labeled)

Use two colons :: to separate the term and the description.

Term 1:: Description of term 1.
Term 2::
+
A more detailed description of term 2 that spans multiple lines.

Checklists

Start with * [ ] for unchecked or * [x] for checked items.

* [ ] Task 1
* [x] Task 2 (completed)

Callout Lists

Used to add annotations to code blocks. Add `// ` style callouts in the code and then create a callout list after the code block.

public class Example { //
    public static void main(String[] args) { //
        System.out.println("Hello"); //
    }
}
Definition of the main class.
The main method.
Prints "Hello" to the console.

List Continuation

Use a + on a new line to attach a new block (like a paragraph or another list) to the preceding list item.

* Item with a paragraph +
This paragraph is part of the previous list item.

* Another item

Links and Cross-references

External Links

Use the link: macro followed by the URL in square brackets [] and optionally the link text. To open in a new tab, add ^ after the URL.

link:https://asciidoctor.org[Asciidoctor Website]
Visit link:https://example.com[] for more info.
link:https://example.com[Example^]

Internal Cross-references

Use [section-id] to link to a section with the specified ID. Section IDs are usually auto-generated. Use before the section title for custom IDs.

== My Section
[[important-info]]
=== Important Information

See <<important-info>> for details.
See <<important-info,Key Details>> for details with custom text.

Cross-references to Other Documents (Asciidoctor)

Use Link Text.

See xref:another-document.adoc#subsection[More in Another Doc]

Inline Anchors

Create a target location within a paragraph using .

This paragraph contains an [[here]]anchor.

...

See the details <<here>>.

Footnotes

Create a footnote using [1] inline, or with explicit markers [2] and [2].

This is some text with a footnote:[Explanation here].
Another way to add a footnote is with a marker.^1^
...
[^1] This is the explanation for the marker.

Images

Block Images

Use the image::[] macro on a new line.

image::path/to/image.png[Alt Text,width=300,align=center]

Inline Images

Use the image:[] macro within a line of text.

Click the image:icons/play.png[Play,title=Start] button.

Tables

Basic Tables

Use pipes | to separate columns and equals signs = to define the table boundaries (|=== at the start and end).

|===
| Header 1 | Header 2
| Cell 1.1 | Cell 1.2
| Cell 2.1 | Cell 2.2
|===

Tables with Titles

Add a title above the |=== block, prefixed with a dot ..

.My Table Title
|===
| Col A | Col B
| 1     | 2
| 3     | 4
|===

Table Options

Use attributes within square brackets [] before |===.

[cols="2*,1*,3",options="header"]
|===
| Name | Value | Description
| Item A | 10    | Some details
| Item B | 25    | More information
|===

[width="80%",align="left,center"]
|===
| Left Aligned | Center Aligned
| Data 1       | Data 2
|===

Cell Formatting

Use formatting characters at the beginning of a cell: >, <, ^, e, b, u, m, l, a.

|===
| <Item | ^Quantity | >Price
| *Product X* | ^10 | >$5.00
| Item Y | 20 | $10.00
|===

Cell Spans

Use .<n>` for column span and `^<n> for row span.

|===
| Name | Details
| 2+^Big Cell | Information here
| Item 2 | More info
| Item 3 | Even more info
| Name | Description
| 2.2+ Multi-span cell | ...
| Item A | ...
| Item B | ...
|===

Admonitions

Standard admonition blocks to highlight important information.

NOTE

NOTE: Single-line note. or [NOTE] ==== Multi-line note. ====.

NOTE: Remember to save your work frequently.

[NOTE]
====
This is a multi-line note.
It contains important information.
====

TIP

TIP: Single-line tip. or [TIP] ==== Multi-line tip. ====.

TIP: Try this helpful shortcut.

[TIP]
====
Here's a useful tip that can save you time.
====

IMPORTANT

IMPORTANT: Single-line important info. or [IMPORTANT] ==== Multi-line important info. ====.

IMPORTANT: This step is crucial for the process.

[IMPORTANT]
====
Failure to complete this step will result in errors.
====

WARNING

WARNING: Single-line warning. or [WARNING] ==== Multi-line warning. ====.

WARNING: Incorrect usage can lead to data loss.

[WARNING]
====
Be very careful when performing this action.
====

CAUTION

CAUTION: Single-line caution. or [CAUTION] ==== Multi-line caution. ====.

CAUTION: The surface may be hot.

[CAUTION]
====
Exercise caution when handling this equipment.
====

Blockquotes

Use [quote] or [verse] followed by ==== to create a blockquote. quote is for general quotations, while verse preserves line breaks.

[quote, Author Name]
____
This is a block quotation.
It can span multiple lines.
____

[verse, Poet]
____
The woods are lovely, dark and deep,
But I have promises to keep,
And miles to go before I sleep,
And miles to go before I sleep.
____

Sidebars

Use [sidebar] followed by ==== to create a sidebar block.

[sidebar]
====
This is a sidebar.
It provides additional context
that is not essential to the main content.
====

Literal Blocks

Used to display text exactly as it is, preserving whitespace and line breaks.

Literal Paragraph

Start a paragraph with a space.

 This is a literal paragraph.
 It preserves the spacing.

Delimited Literal Block

Use [literal] followed by ----.

[literal]

This is a literal block.
